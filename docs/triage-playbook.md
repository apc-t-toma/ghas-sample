# GHAS トリアージプレイブック

## 判定分類

1. True Positive: 到達可能なコード経路で実際に悪用可能
2. Needs Context: 悪用可能性が不明で、オーナー確認が必要
3. False Positive: 明示的な制御により悪用不可能

## トリアージ手順

1. 検出コンテキストを再現する
2. 重大度と影響パスを確認する
3. 修正、抑止、リスク受容のいずれかを決定する
4. 判定根拠を PR とアラートコメントに記録する

## 抑止ルール

1. 抑止は技術的根拠を文書化した場合のみ許可する
2. 再評価期限を必ず設定する
3. 追跡用チケットまたは PR を必ず紐付ける

## 例外運用ルール

1. 例外申請には次を必須とする
   - チケット番号
   - 技術的根拠
   - 代替コントロール
   - 再評価期限
2. 期限切れ例外は再トリアージ対象に戻す
3. 例外承認者は変更実装者と分離する

## SLA 例

1. Critical: 24 時間以内に修正または緩和
2. High: 7 日以内に修正または緩和
3. Medium/Low: 通常スプリント計画で対応

## 監査ログ方針

1. すべての判定と理由を PR または Issue に記録する
2. Suppressed アラートは週次で再確認する
3. High 以上の期限超過は週次レポートで可視化する
